**Phase**: 1 | **Date**: 2026-02-28 | **Plan**: `specs/001-injury-simulation/plan.md`

## Entities

### Player

Sourced from the static FC 25 dataset (`public/data/players.json`).
Read-only at runtime; never mutated by the game.

```typescript
interface Player {
  id: string;             // Transfermarkt ID (cross-referenced during data prep)
                          // Fallback: slugified FIFA name if no TM match
  fifaId: string;         // Row identifier from the FC 25 Kaggle CSV
  name: string;           // Display name (e.g., "M. Salah")
  fullName: string;       // Full name for search (e.g., "Mohamed Salah")
  age: number;            // Age at time of dataset snapshot
  position: Position;     // Primary position
  altPositions: Position[]; // Alternative positions
  teamId: string;         // Slugified team name (e.g., "liverpool-fc")
  teamName: string;       // Display team name
  league: string;         // League name (e.g., "Premier League")
  nation: string;         // Nationality
  overallRating: number;  // Overall rating (0–99)
}

type Position = 'GK' | 'DEF' | 'MID' | 'FWD';
```

### InjuryProfile

Pre-computed per player from Transfermarkt history (`public/data/injury-profiles.json`).
Used to parameterize the Bayesian simulation model.
Read-only at runtime.

```typescript
interface InjuryProfile {
  playerId: string;         // Matches Player.id (Transfermarkt ID or slug)
  observedInjuries: number; // Total injury occurrences in recorded history
  observedSeasons: number;  // Number of seasons of data available (can be fractional)
  meanDaysAbsent: number;   // Mean duration of injuries for this player
  // Derived at compute time; not stored:
  //   posteriorSeasonRate = (3.0 * 2.0 + observedInjuries) / (3.0 + observedSeasons)
}
```

Players with no Transfermarkt match use the league default profile:
```typescript
const DEFAULT_INJURY_PROFILE: InjuryProfile = {
  playerId: '__default__',
  observedInjuries: 2.0,
  observedSeasons: 1.0,
  meanDaysAbsent: 18,
};
```

### InjuryEvent

A single injury occurrence within a game session. Generated by the simulation engine
and stored in the game state.

```typescript
interface InjuryEvent {
  id: string;            // UUID (generated at simulation time)
  playerId: string;      // Matches Player.id
  playerName: string;    // Denormalized for display without lookup
  startDate: string;     // ISO 8601 date (YYYY-MM-DD) when injury occurred
  returnDate: string;    // ISO 8601 date when player becomes available again
  daysAbsent: number;    // Duration of injury
  injuryTier: InjuryTier; // For display/explainability
}

type InjuryTier = 'minimal' | 'moderate' | 'severe';
```

---

### MatchDay

A recorded game event. Drives the simulation: advancing to a match date triggers
injury checks for all players.

```typescript
interface MatchDay {
  date: string;         // ISO 8601 (YYYY-MM-DD)
  opponent: string;     // Optional user-entered opponent name
  injuryEvents: string[]; // InjuryEvent IDs generated on this match day
}
```

---

### GameSession

Represents one campaign (one team, one persistent timeline). Multiple sessions
can coexist (different teams or restarts).

```typescript
interface GameSession {
  id: string;           // UUID
  schemaVersion: number; // For IndexedDB migration (current: 1)
  teamId: string;
  teamName: string;
  createdAt: string;    // ISO 8601 timestamp
  lastPlayedAt: string; // ISO 8601 timestamp
  currentDate: string;  // ISO 8601 date — the "now" of the campaign
  matchHistory: MatchDay[];
  activeInjuries: InjuryEvent[]; // Injuries where returnDate >= currentDate
  resolvedInjuries: InjuryEvent[]; // Injuries where returnDate < currentDate
}
```

### AppState (Zustand store shape)

In-memory state. Serialized to / deserialized from IndexedDB on mount and on
every mutation.

```typescript
interface AppState {
  // Data (loaded from static JSON once at startup)
  players: Player[];          // All players for the selected team
  injuryProfiles: Map<string, InjuryProfile>; // playerId → profile

  // UI state
  view: AppView;
  selectedTeamId: string | null;

  // Session management
  sessions: GameSession[];
  activeSessionId: string | null;

  // Derived (computed, not stored)
  // activeSession: sessions.find(s => s.id === activeSessionId)
  // currentInjuries: activeSession.activeInjuries filtered by currentDate
}

type AppView =
  | 'team-select'     // Initial screen: pick a team
  | 'session-select'  // Load existing session or start new
  | 'injury-board'    // Primary view: current injuries + return dates
  | 'advance-date';   // Enter next match date
```

## State Transitions

```
[team-select]
    ↓ user selects team
[session-select]
    ↓ user starts new session OR loads existing
[injury-board]                    ← primary view
    ↓ user taps "Advance to match"
[advance-date]
    ↓ user enters date + confirms
    → simulation runs for each day between currentDate and new date
    → new InjuryEvents generated
    → GameSession.currentDate updated
    → activeInjuries / resolvedInjuries recomputed
[injury-board]                    ← back to primary view
```

## IndexedDB Schema (version 1)

Database name: `fifa-injury-tracker`

| Object Store | Key | Value | Indexes |
|---|---|---|---|
| `sessions` | `id` (UUID) | `GameSession` | `teamId`, `lastPlayedAt` |
| `playerCache` | `teamId` | `Player[]` | — |
| `profileCache` | `teamId` | `InjuryProfile[]` | — |

`playerCache` and `profileCache` store parsed static JSON per team so that
re-fetching `public/data/players.json` on every load is avoided.

---

## Validation Rules

- `InjuryEvent.returnDate` MUST be >= `InjuryEvent.startDate`.
- `InjuryEvent.daysAbsent` MUST be >= 1.
- `GameSession.currentDate` MUST NOT go backward (no time travel).
- `Player.age` MUST be in range [15, 45].
- `InjuryProfile.observedSeasons` MUST be > 0 (use 1.0 minimum as fallback).
- `GameSession.schemaVersion` MUST match the current schema version; mismatch
  triggers a migration handler before reads/writes proceed.

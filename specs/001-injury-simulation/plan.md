# Implementation Plan: FIFA Injury Simulation

**Branch**: `master (001-injury-simulation)` | **Date**: 2026-02-28
**Spec**: `specs/001-injury-simulation/spec.md` (pending — derived from user description)

## Summary

Build a React (TypeScript) SPA that simulates player injuries for a selected FIFA
(FC 25) team. The user selects a team, enters a match date, and the app calculates —
using a Bayesian injury model grounded in real Transfermarkt data + UEFA epidemiology —
which players are currently injured and when they return. Game state persists across
sessions via IndexedDB so the user can advance match-by-match indefinitely.

Player data is sourced from the FC 25 Kaggle dataset (static JSON). Injury history
is sourced from the Figshare Transfermarkt injuries dataset. Both are bundled as
static assets — no backend required.

## Technical Context

**Language/Version**: TypeScript 5.x + React 18
**Primary Dependencies**: Vite 5, Zustand 4, `idb` 8, `papaparse`, `date-fns`,
  Tailwind CSS 3
**Storage**: IndexedDB via `idb` (game sessions + injury events); `public/data/`
  static JSON for player roster + injury profiles
**Testing**: Vitest + React Testing Library (optional — only if requested per feature)
**Target Platform**: Modern desktop browser (Chrome/Firefox/Safari latest 2 versions);
  static site deployment (Netlify/Vercel/GitHub Pages)
**Project Type**: Frontend SPA
**Performance Goals**: Initial load <3 s; injury simulation advance <200 ms for 25
  players × 30 days = 750 daily checks
**Constraints**: No backend; CORS-compliant (all data from `public/` — same-origin);
  offline-capable after first load; no PII stored or transmitted
**Scale/Scope**: ~25 players per team; ~1–5 active sessions; ~5 primary screens

**Structure Decision**: Single frontend project, source at repo root `src/`.
Simulation logic isolated under `src/simulation/` for independent testability.
Data fetching isolated under `src/data/`. One-time data scripts under `scripts/`
(not bundled with the app).

## Complexity Tracking

| Item | Why Needed | Simpler Alternative Rejected Because |
|------|------------|--------------------------------------|
| IndexedDB (not localStorage) | Campaign state can hold 25 players × 100+ match days of injury history; estimated 1–5 MB per long session | localStorage 5 MB cap would be hit for multi-season campaigns |
| Bayesian shrinkage model | Required by Principle II: individual rates must be grounded in data; pure average ignores player-specific history | Simple average would ignore players with unusually high/low injury rates |
| One-time data prep scripts | Kaggle CSV needs to be converted; player IDs need to be cross-referenced before runtime | Doing this at runtime would be slow (5 MB CSV parse on every load) and violates Principle V |
